v1.2.2
- Fix: Bionic duplicants set to Bypass no longer dock and unequip their atmo / oxygen-mask suits at customised checkpoints when the suit runs low on O2. Vanilla's "return suit when low / idle" chore (SuitLocker.ReturnSuitWorkable) targets the SuitLocker building directly, bypassing the SuitMarker reactables we patch, which is why Bypass appeared to leak after the recent DLC retuning. A new precondition is added to both the urgent and idle return-suit chores, gating each candidate dupe against the locker's paired SuitMarker.
- Fix: per-dupe Block on the side screen now also gates the SuitLocker buildings behind the checkpoint, not just the marker cell. Previously a Blocked dupe could still walk around to the dock side and drop their suit, sometimes getting stuck on the ground if the rack was occupied.
- Safety: Bionics in Bypass keep their atmo suit on indefinitely. If the suit empties with no breathable-air route, they can suffocate — the option dropdown tooltip warns about this and recommends an oxygen pocket or manual unsuit before flipping the mode mid-game.
- Lead Suit and Jet Suit checkpoints are unaffected (no CheckpointAccessControl, precondition is a no-op).

v1.2.1
- Fix: per-duplicant With Suit / Without Suit / Block selections now persist across save and load. Two underlying issues:
    * Storage was List<KeyValuePair<int, bool>>, which KSerialization doesn't round-trip. Replaced with Dictionary<int, bool> (matches vanilla AccessControl's pattern). Fields renamed so v1.2.0 saves don't attempt a type-mismatched deserialization — old v1.2.0 per-dupe overrides are not preserved on upgrade, but they weren't actually persisting before either, so no real data loss.
    * AccessControl and CheckpointAccessControl were attached in a SuitMarker.OnSpawn postfix, which runs AFTER ONI's KSerialization pass. Components were always born with default state on load, dropping any saved data on the floor. Attachment moved to DoPostConfigureComplete patches on each *MarkerConfig (atmo, oxygen mask, lead suit, jet suit), so the components are part of the prefab and present when KSerialization runs.

v1.2.0
- New: in-game options menu (Mods → Better Checkpoints → Options).
- New: "Bionic Duplicants" dropdown (Default / Bypass). Default treats Bionic dupes like Standard at Atmo / Oxygen Mask checkpoints — they appear in the per-checkpoint side screen, equip on entry, and drop on return. Bypass keeps the pre-1.2 behaviour (ignore checkpoint entirely). New default: Default.
- Side screen: Block now participates in a 3-way mutex with With Suit / Without Suit. Exactly one column is checked per dupe at all times; switching columns auto-clears the others.
- Compatibility: changing the dropdown auto-restarts ONI on save. This sidesteps a known crash in Ony's Mod Manager (NRE in `Ony.OxygenNotIncluded.ModManager.ModVisualBuilder.SortButtons`) whose mod-list refresh coroutine holds stale GameObject references after any options dialog save. The auto-restart preempts the coroutine before it can tick.

v1.1.2
- Standard duplicants now ALWAYS drop their suits at the dock on return, regardless of whether the side-screen is set to With Suit or Without Suit mode. The mode toggle now only controls whether they equip on entry.
- Restored strict With Suit / Without Suit mutex (one or the other - never both checked, never neither). Default is With Suit. Use Restrict use to fully block a duplicant.
- Non-Standard duplicants (Bionic, Robot) still ignore the checkpoint entirely - they never equip and never drop, so a bionic wearing an atmo suit walking back through keeps it on.

v1.1.1
- Fix: duplicants now drop their suits at the dock on return (vanilla checkpoint behavior). Default changed from "WithSuit only" (block unequip) to "both modes allowed" so equip-on-entry AND drop-on-exit happen unless overridden.
- Fix: in-game Mods menu shows the actual mod version instead of 0.0.0.0. AssemblyVersion is now stamped into the built DLL.
- UI: relaxed the With Suit / Without Suit mutex. Clicking an unchecked box now only enables that mode (the other stays as it was). Clicking a checked box disables it as long as the other is still checked. Both unchecked is prevented; use Restrict use to fully block a duplicant.

v1.1.0
- Per-duplicant access permissions on Atmo Suit and Oxygen Mask Checkpoints.
- Three columns added to the vanilla Access Permissions side screen for each Standard duplicant:
    * With Suit  - duplicant passes wearing a suit (mutually exclusive with Without Suit)
    * Without Suit - duplicant passes without a suit
    * Restrict use - duplicant is blocked from the checkpoint entirely
- Direction permissions handled by vanilla AccessControl, with full pathfinder integration and automatic re-pathing on toggle.
- Suit-state enforcement at the equip / unequip reactable, not the pathfinder, so dupes still walk past in their current state when a transition isn't allowed.
- Bionic and Robot section headers are hidden on Atmo / Oxygen Mask Checkpoint side screens; non-Standard duplicants ignore those checkpoints entirely (no equip on entry, no drop on exit).
- Lead Suit Checkpoints (and any other vanilla SuitMarker variant) keep fully unmodified vanilla behavior — the existing per-group direction toggles and equip / unequip reactables work as Klei designed them.
- Vanilla-safe and save-safe. Compatible with vanilla and Spaced Out.
