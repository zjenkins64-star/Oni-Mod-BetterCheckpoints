v1.1.2
- Standard duplicants now ALWAYS drop their suits at the dock on return, regardless of whether the side-screen is set to With Suit or Without Suit mode. The mode toggle now only controls whether they equip on entry.
- Restored strict With Suit / Without Suit mutex (one or the other - never both checked, never neither). Default is With Suit. Use Restrict use to fully block a duplicant.
- Non-Standard duplicants (Bionic, Robot) still ignore the checkpoint entirely - they never equip and never drop, so a bionic wearing an atmo suit walking back through keeps it on.

v1.1.1
- Fix: duplicants now drop their suits at the dock on return (vanilla checkpoint behavior). Default changed from "WithSuit only" (block unequip) to "both modes allowed" so equip-on-entry AND drop-on-exit happen unless overridden.
- Fix: in-game Mods menu shows the actual mod version instead of 0.0.0.0. AssemblyVersion is now stamped into the built DLL.
- UI: relaxed the With Suit / Without Suit mutex. Clicking an unchecked box now only enables that mode (the other stays as it was). Clicking a checked box disables it as long as the other is still checked. Both unchecked is prevented; use Restrict use to fully block a duplicant.

v1.1.0
- Per-duplicant access permissions on Atmo Suit and Oxygen Mask Checkpoints.
- Three columns added to the vanilla Access Permissions side screen for each Standard duplicant:
    * With Suit  - duplicant passes wearing a suit (mutually exclusive with Without Suit)
    * Without Suit - duplicant passes without a suit
    * Restrict use - duplicant is blocked from the checkpoint entirely
- Direction permissions handled by vanilla AccessControl, with full pathfinder integration and automatic re-pathing on toggle.
- Suit-state enforcement at the equip / unequip reactable, not the pathfinder, so dupes still walk past in their current state when a transition isn't allowed.
- Bionic and Robot section headers are hidden on Atmo / Oxygen Mask Checkpoint side screens; non-Standard duplicants ignore those checkpoints entirely (no equip on entry, no drop on exit).
- Lead Suit Checkpoints (and any other vanilla SuitMarker variant) keep fully unmodified vanilla behavior — the existing per-group direction toggles and equip / unequip reactables work as Klei designed them.
- Vanilla-safe and save-safe. Compatible with vanilla and Spaced Out.

v1.0.0
- Initial scaffold (per-checkpoint toggles, custom side screen). Superseded by 1.1.0.
